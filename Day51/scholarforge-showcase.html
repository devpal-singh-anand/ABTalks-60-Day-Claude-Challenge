The Problem

Two mechanical tasks, none of it thinking.

Students and independent researchers lose disproportionate time converting notes into publication-style prose, then reformatting that prose to satisfy a citation standard. Nothing on the market solves both without a cost.

System Architecture

Seven layers. One rule that governs all of them.

A layered, offline-first monolith — not a distributed system. One user, one process, one 4GB GPU. Ports & adapters shape, so V2 additions plug in behind existing interfaces.

"The AI Generation Layer and the Deterministic Formatting Layer never share responsibility for the same output. The model only ever produces language — every margin, heading, and citation mark is computed by code that has no dependency on model behavior."

01 · PRESENTATIONNotes Editor · Template Picker · Draft & Reviewer View
Owns rendering and input capture. Never touches business logic, formatting math, or the model directly.
02 · APPLICATIONWorkflow Orchestrators
Owns use-case orchestration — generate, review, save, export — and dependency wiring. Never knows how a layer works internally.
03 · AIPromptBuilder · ModelClient · OutputCleaner · ReviewerEngine
Owns prompting, inference, voice injection, cleanup. Never decides a margin, heading, or file format.
04 · FORMATTINGTemplate Engine · Style Rules
Owns per-standard layout and citation structure. Never generates prose or calls the model.
05 · PERSISTENCESession Store · Autosave
Owns serialization, autosave, load-with-recovery. Never makes formatting or AI decisions.
06 · EXPORTDOCX Builder
Owns assembling the final file from what it's given. Never decides what the content should be.
07 · CONFIGURATIONModel · Voice · Template Registry · Paths
Owns every tunable in the system. Never contains runtime business logic.

The 10-Day Build

Each day adds one working layer — never a promise of one.

Day 2 through Day 10, in the order the architecture actually gets built, ending in a stable, cold-start-demoable Version 1.0.

DAY 2
Project skeleton + Notes Module: the structured research notes editor, zero AI or formatting dependencies yet.
DAY 3
Configuration Layer + AI connectivity skeleton (IModelClient, Ollama & llama.cpp). Answers the riskiest question: does Qwen2.5‑3B run acceptably on 4GB?
DAY 4
Full prose-generation pipeline: PromptBuilder, VoiceProfile, OutputCleaner, GenerationWorkflow. Notes → generated prose becomes real.
DAY 5
Data-driven Template Engine + the first complete TemplateDefinition: IEEE (US Letter). Depth over breadth, exactly as scoped.
DAY 6
Export Layer: DocxExporter/DocxRenderer. Notes → Generation → IEEE Formatting → DOCX works end-to-end for the first time.
DAY 7
Session/Persistence Layer: manual save, autosave, reload-with-recovery. Nothing survives an app restart by accident — it's engineered to.
DAY 8
Reviewer Mode end-to-end, plus a dedicated error-handling hardening pass across every layer built so far.
DAY 9
Stretch: template coverage beyond IEEE — APA (6th, 7th), MLA (8th, 9th), Chicago (17th, 18th). Additive, not required to hit Day 10.
DAY 10
Integration, polish, full regression, live demo rehearsal. No new features — just a submittable Version 1.0.
Success Metrics

How Day 10 gets judged.

✓

At least one full template standard works end-to-end with zero manual reformatting after export.

✓

100% of the generation-to-export pipeline runs offline, with no network calls at any point.

✓

Session save/load succeeds with no data loss across repeated test cycles.

✓

Reviewer Mode output reads as content-specific, not generic, in manual review of test drafts.

✓

Generated text is recognizably distinct from unedited, generic AI output once the voice profile is applied.

✓

The system survives a live, cold-start demo without failure.

Known Risks

Named early, not discovered on Day 9.

Model Quality

Qwen2.5‑3B is small; generated prose may need heavier prompt scaffolding to sound competent and on-voice.

Hardware

4GB VRAM is tight for a 3B model at reasonable context length — needs empirical testing, not assumption.

Scope

Six citation-standard/edition combinations is a lot of deterministic logic to finish correctly in 10 days.

Style Preservation

Voice preservation via prompting is inherently approximate — success criteria stay honest about that.

Timeline

Solo 10-day builds fail most often on integration and polish time, not core feature time.
