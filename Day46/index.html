What is this?

A production-grade autonomous multi-agent orchestration platform. Eight specialized AI agents collaborate in a closed feedback loop to produce a competitive intelligence research report — with zero human intervention after kickoff.

The 8 Agents

  • Planner — runs once. Decomposes the objective, sets strategy, defines evaluation goals.
  • Executor — every round. Drafts / extends the research output.
  • Evaluator — every round. Round 1 generates a dynamic rubric; all rounds score 0–100.
  • Critic — every round. Surfaces weaknesses, gaps, unsupported claims.
  • Improver — every round. Produces a strictly better version from eval + critique + memory.
  • Memory Manager — local state; AI only for periodic summarization.
  • Safety Monitor — local validation; AI only for flagged content.
  • Final Reviewer — runs once. Produces the executive-ready final report.

Information Flow

Planner seeds strategy → Executor drafts → Evaluator scores → Critic critiques → Improver refines → loop. Each iteration threads previous outputs forward as context, so the system compounds knowledge rather than restarting.

Feedback Loop

A circular cycle: Improver → Evaluator → Critic → Improver. The loop only exits when a stop condition fires. Open-ended by design — the system decides itself when "good enough".

Dynamic Rubric

Round 1: the Evaluator autonomously decides which dimensions matter for the task (accuracy, completeness, depth, actionability, evidence, clarity…), assigns weights, and justifies each. The same rubric is then used every subsequent round for consistent scoring.

Stop Conditions

  • Target Quality — score ≥ 85 ⇒ success.
  • Plateau — Δ < 3 for 2 consecutive rounds ⇒ converged.
  • Safety — unsafe output ⇒ halt immediately.
  • Max Iterations — 10 (safety fallback only).

Production Architecture

Adapter pattern for providers — adding a new vendor means implementing one adapter, nothing else changes. Agents are stateless functions over a shared state object. Retries with exponential backoff, timeout handling, and rate-limit detection are built in.

Why autonomous?

Removes the human bottleneck. The system iterates, self-critiques, and self-improves until it crosses a quality bar — not until a timer expires. Same pattern that powers production agent frameworks (AutoGen, LangGraph, CrewAI) at scale.
